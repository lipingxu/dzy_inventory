# 图书资产管理系统 (Inventory Management System)

## 项目简介
该系统已从单纯的“价格追踪器”升级为“**图书资产仓库**”。它能让你管理自己的购入/售出记录，并自动计算每本书的实际盈亏，同时实时监控多抓鱼的回收行情。

## 核心文件
*   `sync_data.py`: **唯一运行脚本**。整合了数据抓取、数据库更新和报表生成功能。
*   `inventory.csv`: **核心数据库**。包含：ISBN、书名、状态、购入价格、售出价格、历史最高价及近 7 天行情。
*   `report.html`: 可视化管理报表。

## 仓库操作指南

### 1. 极速更新行情 (核心操作)
此方案利用 macOS 剪贴板，彻底绕过多抓鱼的反爬风控，无需保存文件：
1.  **在 Chrome 中复制**：
    *   打开多抓鱼待售页面并刷新。
    *   在开发者工具 `Network` 面板搜索 `inquiry`。
    *   **右键**点击 `inquiry-books` 请求 -> **`Copy`** -> **`Copy response`**。
2.  **运行同步脚本**：
    *   在终端输入：`python3 sync_data.py`。
    *   脚本会自动读取剪贴板，解析并输出实时涨跌摘要。

### 2. 录入购入价格 / 标记已售
*   **购入价**：在 `inventory.csv` 的 `购入价格` 列填入成本。
*   **标记已售**：将 `状态` 改为 `已售`，并在 `售出价格` 列填入金额。重新运行脚本后，该书将移至报表的“已售结项”区。

## 报表视觉逻辑
*   **当前库存**：
    *   **排序规则**：严格遵循抓包数据中的原始顺序。
    *   **价格预警**：当前价格低于历史最高价时，价格显示为**绿色**。
    *   **近7天趋势**：红色向上箭头 (↑) 表示涨，绿色向下箭头 (↓) 表示跌，横线 (-) 表示无变化。
    *   **灰色整行**：多抓鱼当前不收的书籍（回收价为0）。

## 核心机制说明
*   **智能资产管理 (自动清理与平仓)**：
    *   **资产保护**：只要填过 `购入价格` 或 `售出价格`，该书即被视为“正式资产”，无论多抓鱼是否下架，都会永久保留并显示在报表中。
    *   **草稿清理**：未填入任何价格且在最新抓包中消失的书，视为“随便看看”的草稿，系统会自动将其设为 `已移除` 并从 HTML 报表中隐藏。
    *   **自动平仓**：在 `inventory.csv` 中填入 `售出价格` 后，脚本运行会**自动**将状态转为 `已售`。
*   **历史最高价 (只升不降)**：永久记录。系统会自动比对 `MAX(旧最高价, 最新价)`，不会随行情下跌而减小。
*   **数据保护**：用户手动填写的 `购入价格`、`售出价格` 这两列由用户完全掌控，脚本绝不覆盖。
*   **日期归一化**：自动将所有日期统一为 `YYYY-MM-DD` 格式。
